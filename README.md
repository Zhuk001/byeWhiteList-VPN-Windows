# byeWhiteList VPN (Windows)

`byeWhiteList` — WPF‑клиент для запуска `xray.exe` в TUN‑режиме на Windows с маршрутизацией на основе `geosite.dat/geoip.dat` (DIRECT для выбранных правил, остальное через VPN).

## Скачать

Релизы и обновления публикуются в GitHub Releases:
- https://github.com/Zhuk001/byeWhiteList-VPN-Windows/releases

## Варианты поставки

### 1) Портативная версия (рекомендуется)
- Скачайте `*.zip`, распакуйте в любую папку и запустите `byeWhiteList.exe`.
- В архиве уже лежит всё необходимое (включая `xray.exe` и geodata) — ничего дополнительно качать не нужно.

### 2) Установщик
- Скачайте `Setup.exe`/`MSI` и установите как обычную программу.

Данные пользователя (серверы, избранное, правила и т.д.) хранятся в `%LOCALAPPDATA%\ByeWhiteList\` и **не удаляются** при обновлениях.

## Обновление приложения

В приложении есть окно обновления:
- `Настройки` → `Обновление приложения`

Логика:
- приложение запрашивает `latest release` на GitHub;
- если версия релиза выше текущей — показывает версию, описание релиза и кнопки:
  - зелёная `Обновить`
  - красная `Отмена`
- при нажатии `Обновить` архив скачивается в фоне, обновление применяется и приложение перезапускается.

Примечание: авто‑обновление рассчитано на портативные `.zip` релизы. Если программа установлена в защищённую папку (например, `Program Files`) и нет прав на запись, обновление будет недоступно — используйте портативную версию.

## Маршрутизация

Используется логика:
- `tun-in` → по умолчанию `proxy`
- выбранные правила `geoip/geosite` → `direct`
- ручные исключения:
  - вкладка `DIRECT` — всегда мимо VPN
  - вкладка `VPN` — всегда через VPN

## Безопасность (про localhost‑прокси)

В некоторых клиентах встречается проблема: локальный SOCKS/HTTP‑прокси на `127.0.0.1` запускается **без авторизации**, и стороннее ПО на компьютере может подключиться к нему и вычислить внешний IP прокси.

В `byeWhiteList`:
- постоянный локальный SOCKS‑прокси **по умолчанию отключён** и не запускается;
- Speedtest использует локальный HTTP‑прокси **со случайными учётными данными** (Basic Auth) на каждый запуск.

## Требования

- Windows 10/11.
- Права администратора нужны для добавления/удаления маршрутов и DNS override.
